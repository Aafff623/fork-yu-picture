# ADR-0001：身份迁移保持上游可追溯

- 状态：Accepted
- 日期：2026-07-18
- 决策者：threetwoa

## 背景

仓库需要作为独立二开维护，同时保留原项目许可证与来源。直接删除所有上游痕迹会损害可追溯性；保留源码内作者营销和旧包名又会造成维护身份混乱。

## 决策

1. 自有包名、作者元数据、示例身份和产品维护者迁移为 `threetwoa`。
2. README 的 Upstream/致谢和 Git remote 保留原始来源。
3. 第三方许可证、依赖坐标和协议常量不因品牌迁移而修改。
4. 迁移必须覆盖目录、声明、imports、Mapper/SPI/扫描配置，并通过编译或目标构建验证。
5. 营销导流从产品和教学主链路移除。

## 后果

优点：维护边界清晰、可继续同步上游、许可证链完整。代价：未来合并 upstream 时包路径冲突需要人工解决，不能直接无脑 merge。
