# Delivery Contract

每次交付必须回答：

| 项目 | 要求 |
|---|---|
| Outcome | 用户现在能做什么，或什么风险被消除 |
| Scope | 修改了哪些模块，明确未修改部分 |
| Behavior | 业务行为是否变化；若变化给出验收路径 |
| Verification | 命令、通过项、跳过项和环境阻塞 |
| Risk | 数据、兼容性、外部服务与回滚风险 |
| Rollback | 可安全回退的提交或配置开关 |
| Follow-up | 不阻塞本次交付但应继续处理的事项 |

截图、日志和大文件只在确有验收价值时保存，且不得包含凭据或个人数据。
